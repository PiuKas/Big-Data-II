Big Data y Razzmatazz: un estudio sobre sus playlists de Spotify

1. Introducción

Razzmatazz es una sala de las salas de conciertos más importantes de Barcelona y no solo eso, es una de las discotecas de referencia en la ciudad condal. Esto es así gracias a sus características y a todas las temáticas y salas de las que dispone cada día de la semana. De las más destacadas y conocidas son El Dirty, Mandanga, la 1 de Razzmatazz y Fuego. El objetivo de esta investigación es analizar las características de las canciones que conforman las playlist de Spotify (creadas por Razzmatazz) de las tres fiestas mencionadas anteriormente. Con la API de Spotify, Spotipy, se usará el Get Audio Features para recolectar datos de las canciones como son la bailabilidad, energía o tempo entre otros (de las tres playlists) y luego comparar estos parámetros para extraer conclusiones.

1.1. Las playlists

El Dirty

El Dirty es la fiesta temática de los miércoles que define la música que pone como Hits, Dance Pop, Reggaeton y Mamarracheo. Es una noche especialmente LGBTIQ+ friendly que todas las semanas cambia su ambientación. Su playlist, a fecha de 23 de mayo del 2023, está encabeza por la Bizarrap Sesión de Shakira.

La 1

La 1 es la sala principal de las fiestas de los sábados. El protagonsita es el pop acompañado de estilo urbano y clásicos. Es una de las salas con más música variada, tal y como refleja su playlist en la que aparecen artistas muy diferentes entre si como Natalia Lacunza, Lizzo, Drake o Elthon John.

Mandanga

Mandanga, la reina de los jueves universitarios, deja que los hits, el pop y el perreo dominen su fiesta. Es una noche en la que solo está permitido bailar y pasárselo bien. La lista esta conformada por música diversa llena de clásicos y es de las playlists con más likes del perfil de razzmatazzclubs.

Fuego

Los viernes son de la sala Fuego que representa la música urbana en su esencia llenando la discoteca de hip hop, reggaeton, drill o afrobeats entre otros.

A continuación se procede a comenzar con la extracción de canciones de la playlist y se va comentando el código línea a línea.

def query_principal(p, offset, contador):  #Se define una función para crear un documento JSON por cada 100 canciones de cada playlist
        query = sp.playlist_items(p, fields=None, offset=offset, market=None) #Se llama a la query o función
        if query['next'] == None:  #Si la query no encuentra la variable "next" quiere decir que ya ha recogido todas las canciones de la playlist 
            with open(f'api-res/playlist-{p}-{contador}-response.json', 'w', encoding='UTF-8') as f:  #Se abre un archivo JSON 
                json.dump(query, f, ensure_ascii=False, indent=4)  #Se mete la query dentro del json
            pass  
        else:  #Si todavia quedan canciones en la playlist
            with open(f'api-res/playlist-{p}-{contador}-response.json', 'w', encoding='UTF-8') as f:  #Se abre un archivo JSON
                json.dump(query, f, ensure_ascii=False, indent=4)  #Se mete la query dentro del JSON
            contador = contador+1  
            offset = offset+100  #Se suma 100 puntos al offset
            time.sleep(1)  
            query_principal(p, offset, contador)  #Se vuelve a lanzar la función con el offset aumentado para que la respuesta comience 100 canciones después de la última respuesta
    
    for p in playlists:  #Se crea un bucle que recorra la lista de playlist
        query_principal(p, offset, contador) #Se lanza la función

Una vez extraidas las canciones, hay que extraer las variables de cada una para analizarlas. En este caso van a ser: nombre, artista, popularity e id de cada canción

files = glob.glob("api-res/playlist-*.json")  #Se abren todos los JSON que se han creado
llista_cancons = [] #Se crea una lista en la que recoger todos los datos de las canciones

for file in files: #Se crea una función que recorra todos los archivos JSON
	playlist_id = file.split("-")[2] #Se extrae el id de la playlist del nombre del archivo
	print(playlist_id)
	f = open(file, encoding="utf8") #Se abre el JSON
	data = json.load(f) #Se carga el JSON
	tracks = data["items"] 
	for track in tracks: #Se recorre cada canción dentro del JSON y se sacan las variables necessarias
		playlist = playlist_id 
		name = track['track']['name']
		art_name = track['track']['artists'][0]["name"]
		t_id = track['track']['id']
		popularity = track['track']['popularity']
		tup = ({"playlist_id":playlist, "id":t_id, "name":name, "artists_name":art_name, "popularity":popularity}) #Creamos una tupla con un diccionario con los datos que queremos extraer
		llista_cancons.append(tup) #Se guarda la tupla en la lista de canciones

with open(f'data.json', 'w', encoding='UTF-8') as f: #Se guardan todos los datos en un JSON
	json.dump(llista_cancons, f, ensure_ascii=False, indent=4)

Por último, se extraen las features y se crea el DataFrame

f = open('data.json', encoding="utf8")  #Se abre el JSON con los datos de todas las canciones
data = json.load(f)  #Se carga el JSON 
  
total_data = []  #Se crea una lista para llenarla con los datos
  
for d in tqdm(data): #Se recorre el JSON canción por canción
    features = sp.audio_features(d['id']) #Se utiliza la la id de la canción para extraer los features
    d["features"] = features #Se crea una variable de Dataframe con las features
    track_df = pd.DataFrame.from_dict(d, orient="columns") #Se crea un Dataframe con la información de las canciones del JSON 
    features_data = pd.DataFrame.from_dict(features, orient='columns') #Creamos un Dataframe con las features de cada canción   
    final_track_df = pd.concat([track_df, features_data], axis=1) #Se unen los dos Dataframes en uno
    total_data.append(final_track_df) #Se guarda este Dataframe en la lista de Dataframes
    time.sleep(1)  
total_df = pd.concat(total_data)  #Se unen todos los Dataframes en uno resultante
total_df.to_csv("total_df.csv", index=False) #Se convierte el Dataframe final en csv
